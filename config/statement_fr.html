Brandubh

Un ancien jeu de stratégie celtique. Un petit groupe de défenseurs doit aider son Roi à s'enfuir, tandis qu'une large force d'attaquants tente de le capturer.

  L'Objectif

Incarnez soit les Attaquants soit les Défenseurs dans une partie de Brandubh (une variante du Hnefatafl). Vous devez remplir vos conditions de victoire avant votre adversaire.

Note : Les Défenseurs contrôlent le Roi ainsi que les pièces de défense standard.

  Les Règles

Pour tous les détails, vous pouvez consulter les règles officielles au format PDF. Notez cependant ces quelques différences avec l'adaptation CodinGame :

  • Les règles officielles parlent de pièces "Noires" et "Blanches" au lieu "d'Attaquants" et de "Défenseurs".
  • Le point 10 (une partie complète se joue en deux manches) n'est pas applicable. Chaque match sur CodinGame consiste en une seule manche.
  • Le concept de limite de tours n'existe pas dans les règles officielles, mais est utilisé ici pour éviter les parties infinies.

Plateau et Coordonnées : Le jeu se déroule sur un plateau de 7x7 cases. Chaque partie commence depuis la position standard initiale telle que décrite dans les règles officielles. Le système de coordonnées commence à (0, 0) dans le coin supérieur gauche. L'axe X s'étend vers la droite, et l'axe Y vers le bas.

Déplacements : Les Attaquants jouent toujours en premier. Toutes les pièces se déplacent de façon orthogonale (comme la tour aux échecs) d'autant de cases vides qu'elles le souhaitent. Les pièces ne peuvent pas sauter par-dessus d'autres pièces.

Le Trône et les Coins : La case centrale (3,3) est le Trône. Les 4 cases dans les coins sont les zones d'évasion. Seul le Roi peut s'arrêter sur le Trône ou dans les coins. Les autres pièces peuvent passer au-dessus du Trône s'il est vide, mais ne peuvent pas s'y arrêter.

Pièces Hostiles : Les pièces de l'adversaire sont considérées comme des pièces hostiles. Le Trône et les quatre coins du plateau agissent également comme des pièces hostiles pour faciliter les captures.

L'Hostilité du Trône : Le Trône est toujours hostile pour les Attaquants. Cependant, il n'est hostile pour les Défenseurs que si le Roi ne se trouve pas dessus. Le Trône n'est jamais hostile au Roi.

Capture classique : Une pièce (attaquant ou défenseur) est capturée si elle est prise en tenaille sur deux côtés opposés par deux pièces hostiles.

Capture du Roi : Le Roi est capturé comme n'importe quelle autre pièce, sauf lorsqu'il est sur le Trône. S'il est sur le Trône, il doit être entouré sur les 4 côtés pour être capturé.

Fin de Partie : La partie se termine immédiatement lorsqu'une condition de victoire ou de défaite est atteinte. Si aucun camp ne gagne après 200 tours (100 manches pour chaque joueur), un match nul est déclaré.

Code Source : Le code source de ce jeu (Referee) est public et disponible sur GitHub.

Conditions de Victoire
  • Attaquants : Le Roi est capturé, ou tous les défenseurs restants sont encerclés et ne peuvent plus bouger.
  • Défenseurs : Le Roi atteint l'une des 4 cases de coin.
Conditions de Défaite
  • Vous n'avez plus aucun mouvement valide possible.
  • Vous répétez la même position exacte du plateau 3 fois (règle des 3 répétitions).
  • Votre programme fournit une action invalide.
  • Votre programme dépasse la limite de temps.

  Entrées du Jeu

Entrées pour un tour de jeu

Ligne 1 : role - un entier représentant votre rôle (0 pour les Attaquants, 1 pour les Défenseurs).

Les 7 lignes suivantes : Une chaîne de 7 caractères représentant l'état du plateau. Les caractères peuvent être :

  • . : Case vide
  • A : Attaquant
  • D : Défenseur
  • K : Le Roi
Sortie
Une seule ligne contenant votre mouvement au format : x1 y1 x2 y2.
  • x1, y1 : Les coordonnées de départ de la pièce que vous souhaitez déplacer.
  • x2, y2 : Les coordonnées d'arrivée de cette pièce.
Par exemple : 3 1 3 0 pour déplacer la pièce de la case (3,1) vers la case (3,0).
Contraintes
Temps de réponse maximum alloué : ≤ 100 ms pour le tout premier tour, et ≤ 50 ms pour les tours suivants.
